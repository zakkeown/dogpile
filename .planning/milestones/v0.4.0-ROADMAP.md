# Milestone v0.4.0: Recursive Coordination

**Status:** SHIPPED 2026-05-01
**Phases:** 1-5
**Total Plans:** 22
**Requirements:** 27/27 complete

## Overview

Dogpile v0.4.0 lets a `coordinator` run dispatch whole sub-missions (`sequential`, `broadcast`, `shared`, or another `coordinator`) as first-class `delegate` decisions. The release shipped embedded child traces, recursive replay, parent-to-child budget/cancel/cost propagation, bounded child concurrency with local-provider clamping, live stream child-event bubbling, child failure escalation, documentation, examples, changelog, and npm publication.

## Milestone Summary

**Key Accomplishments:**

- Added the public `delegate` decision variant for coordinator-driven recursive sub-runs.
- Added `sub-run-*` event variants, embedded child traces, recursive replay, and accounting recompute.
- Propagated abort, timeout, cost, and token accounting across recursive run trees.
- Added `metadata.locality`, `maxConcurrentChildren`, queued/clamped child events, and local-provider concurrency safety.
- Added live stream child-event bubbling via `parentRunIds` plus coordinator child-failure context and terminal error discrimination.
- Published recursive coordination docs, reference docs, runnable example, changelog, tag `v0.4.0`, GitHub Release, and npm package `@dogpile/sdk@0.4.0`.

**Key Decisions:**

- `delegate` lives on the existing `coordinator` protocol instead of adding a fifth protocol.
- Child traces are embedded inline in parent traces, and `replay()` walks those traces without invoking providers.
- Child event bubbling is live-stream only; stored parent traces and embedded child traces remain ancestry-chain-free.
- `parentRunIds?: readonly string[]` is the canonical stream ancestry shape.
- Local providers clamp child concurrency to 1 and emit a single clamped event per run.
- Terminal unhandled real child failures rethrow the original child `DogpileError` where possible.

**Issues Resolved:**

- Recursive coordination became observable and replayable instead of requiring callers to stitch child runs externally.
- Parent budget ceilings, cancellation, and accounting now compose over nested runs.
- Local model providers are protected from unsafe coordinator fan-out by default.
- Child failures now reach coordinator decision context instead of being opaque terminal errors only.

**Issues Deferred:**

- Caller-defined-tree API: `Dogpile.nest({ children: [...] })`.
- Dedicated `"recursive"` protocol identity.
- Cross-protocol shared transcript across parent/child boundaries.
- Per-child retry policy on delegate decisions.
- OTEL/tracing bridge for sub-run spans.
