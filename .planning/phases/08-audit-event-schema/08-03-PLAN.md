```markdown
### Phase 8: Audit Event Schema

**New API — `@dogpile/sdk/runtime/audit` subpath:**
- `createAuditRecord(trace: Trace): AuditRecord` — pure function that derives a versioned, schema-stable audit record from any completed trace. Works on live `RunResult.trace` and stored/replayed traces.
- `AuditRecord` — exported type, independent of `RunEvent` schema. Contains: `auditSchemaVersion`, `runId`, `intent`, `startedAt`, `completedAt`, `protocol`, `tier`, `modelProviderId`, `agentCount`, `turnCount`, `outcome`, `cost`, `agents`, `childRunIds?`.
- `AuditOutcome` — `{ status: "completed" | "budget-stopped" | "aborted"; terminationCode?: string }`. `terminationCode` carries `BudgetStopReason` values (`"cost" | "tokens" | "iterations" | "timeout"`) when status is `"budget-stopped"`.
- `AuditCost` — `{ usd: number; inputTokens: number; outputTokens: number }`.
- `AuditAgentRecord` — `{ id: string; role: string; turnCount: number }`.
- `AuditOutcomeStatus` — `"completed" | "budget-stopped" | "aborted"`.

**New frozen fixture:** `src/tests/fixtures/audit-record-v1.json` — shape must be updated explicitly when `AuditRecord` schema changes.

**Note:** `AuditRecord` is NOT auto-attached to `RunResult`. Callers invoke `createAuditRecord(result.trace)` explicitly.
```
