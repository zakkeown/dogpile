  const result = await run({
    intent: "Test audit record shape",

**Test case (single it block):**
```typescript
describe("audit record shape contract", () => {
  it("verifies the frozen audit record shape fixture", async () => {
    const raw = await readFile(fixturePath, "utf8");
    const saved = JSON.parse(raw) as Record<string, unknown>;
    const live = await captureAuditRecord();

    // Key presence (AUDT-02 — AuditRecord is schema-stable)
    expect(saved["auditSchemaVersion"]).toBe("1");
    expect(live.auditSchemaVersion).toBe("1");

    // Key ORDER is frozen — any field reordering in createAuditRecord breaks this
    expect(Object.keys(live)).toEqual(Object.keys(saved));

    // Type shape is frozen — any field type change breaks this
    expect(typeShape(live)).toEqual(typeShape(saved));

    // Structural content (objectContaining — values differ between runs)
    expect(saved).toEqual(
      expect.objectContaining({
        auditSchemaVersion: "1",
        runId: expect.any(String),
        intent: expect.any(String),
        startedAt: expect.any(String),
        completedAt: expect.any(String),
        protocol: expect.any(String),
        tier: expect.any(String),
        modelProviderId: expect.any(String),
        agentCount: expect.any(Number),
        turnCount: expect.any(Number),
        outcome: expect.objectContaining({ status: expect.any(String) }),
        cost: expect.objectContaining({
          usd: expect.any(Number),
          inputTokens: expect.any(Number),
          outputTokens: expect.any(Number),
        }),
        agents: expect.any(Array),
      })
    );
  });
});
```

NOTE: The test checks `Object.keys(live).toEqual(Object.keys(saved))` — this enforces that
the implementation assembles fields in the same order as the fixture JSON. If `childRunIds`
is absent in the live run (coordinator with no delegates), the key comparison will show a
mismatch. In that case, adjust: either (a) use a test run that produces a delegate/sub-run,
or (b) compare only the keys that both objects share (exclude optional keys).

If the live coordinator run does not produce a childRunIds field, remove `childRunIds` from
the fixture JSON and update the type-check.ts accordingly, then document the change in the
task summary. The fixture documents what a real run produces.
