   ## v0.4.0 — Recursive Coordination

   ### Added — coordinator delegate decision (Phase 1)
   - `AgentDecision` is now a discriminated union: `ParticipateAgentDecision | DelegateAgentDecision`. Coordinator agents can return `{ type: "delegate", protocol, intent, model?, budget? }` to dispatch a sub-mission.
   - `protocol` ∈ `"sequential" | "broadcast" | "shared" | "coordinator"` for delegate decisions.
   - `maxDepth` config option on engine + per-run (default 4); per-run can only LOWER the engine ceiling. Overflow throws `DogpileError({ code: "invalid-configuration", detail: { reason: "depth-overflow" } })`.
   - Sub-run events on parent trace: `sub-run-started`, `sub-run-completed` (carries embedded child trace), `sub-run-failed` (carries child error). `recursive: true` flag on `sub-run-started` when both parent and child protocol are `coordinator`.
   - `Dogpile.replay(parentTrace)` recurses into embedded child traces; reproduces output, accounting, and event sequence without re-executing children.

   ### Added — budget, cancellation, cost roll-up (Phase 2)
   - Parent abort propagates to all in-flight children via per-child `AbortController`. Children surface `DogpileError({ code: "aborted", detail: { reason: "parent-aborted" } })`.
   - Parent `budget.timeoutMs` is a hard ceiling for the whole tree. Children inherit parent's remaining time as default; per-decision `budget` overrides that exceed remaining are clamped via the new `sub-run-budget-clamped` event.
   - `defaultSubRunTimeoutMs` engine option for legacy callers.
